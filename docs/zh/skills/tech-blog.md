# tech-blog

撰写带源码分析的技术博客。

## 使用场景

- 解释系统内部原理
- 源码分析和走读
- 对比不同实现
- 架构深度解析

## 文档结构

```markdown
# [主题] 深度解析

> **代码版本**：基于 [项目] `vX.Y.Z`

## 1. 引言
## 2. 背景
## 3-N. 核心内容（按数据流组织）
## N+1. 对比 / 权衡
## N+2. 代码索引
## 参考文献
```

## 核心原则

### 渐进式解释
- 从问题开始，而非解决方案
- 逐层构建概念
- 解释"为什么"而非仅仅"是什么"

### 概念前置
- 永远不要使用未定义的术语
- 先定义再使用
- 添加导航提示

### 整体图景优先
- 从视觉概览开始
- 多种方案使用对比图
- 在一张图中展示完整流程

### 平衡对比
- 分析双方
- 使用对比表格
- 识别真正的差异

## 写作指南

### 代码示例
- 包含文件路径和行号
- 解释它做什么
- 使用流程图 + 关键代码片段

### 标注
- 💡 **要点**：关键见解
- ⚠️ **注意**：常见错误
- 📝 **术语**：定义

## 数据完整性

**永远不要捏造**：
- ❌ "压缩比是 16:1"（没有来源）
- ✅ "根据 [来源]，比例是 X:Y"

**需要引用**：
- 性能基准
- 内存使用数据
- 定量对比
