# spec-interview

深度访谈以完善技术规格说明。

## 使用场景

- 完善 `plan.md` 或其他规格文档
- 通过系统性提问细化需求
- 涵盖技术实现、UI/UX、边缘情况、风险考量

## 触发词

- "帮我完善这个 plan"
- "问我关于项目的问题"
- 草稿规格需要全面细化时

## 工作流程

1. 阅读 `plan.md` 或用户指定文档
2. 使用 `AskUserQuestionTool` 进行深度访谈
3. 持续提问直到访谈完成
4. 将完善后的规格写入文件

## 访谈范围

- 技术实现细节
- UI 与 UX 设计
- 潜在顾虑和风险
- 权衡取舍
- 架构决策
- 边缘情况处理

## 原则

- 提出有深度的问题，而非显而易见的问题
- 保持深度，持续探究
- 直到访谈完全结束才停止
